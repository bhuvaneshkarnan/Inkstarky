# Product Requirements Document
## Inkstarky Tattoo — Studio Website

**Prepared for:** Stark Hill Cruz Salonga, Inkstarky Tattoo
**Location:** Dagupan City, Pangasinan
**Document version:** 1.0
**Status:** Draft for build

---

## 1. Overview

### 1.1 Purpose
Inkstarky Tattoo needs a website that functions as the studio's primary booking and portfolio channel, replacing reliance on Facebook cover photos and DMs. The site must look like it was made by a working black & grey realism/horror tattoo artist — not generated by a template or an AI website builder.

### 1.2 The core design risk (read this before building anything)
Most "AI-built" studio sites converge on the same visual language regardless of brief:
- Poppins / Montserrat / Inter / Roboto for everything
- Centered hero, big rounded gradient button, soft drop shadows on every card
- 3-column icon-boxes with rounded corners and pastel tints
- Generic stock "tattoo gun" or "ink splatter" PNGs with drop shadows
- Purple/blue gradients regardless of brand

**This PRD explicitly forbids that output.** Section 4 defines the actual visual system to build. Any implementation (human or AI-assisted) must be checked against Section 4.6 (Anti-Generic Checklist) before it ships.

### 1.3 Success criteria
- A stranger looking at the site for 3 seconds should think "tattoo studio," not "SaaS landing page."
- Typography and layout should look intentionally art-directed, referencing tattoo flash sheets, horror poster design, and editorial print — not dashboard/app UI conventions.
- Booking inquiries increase measurably vs. current Facebook-only funnel.
- Site loads fast and reads well on mobile, since most traffic will come from Facebook/Instagram referral on phones.

## 2. Users & Use Cases

| User | Goal |
|---|---|
| Prospective client (cold, found via IG/FB) | Quickly judge if Stark's style matches what they want, see healed/finished work, understand pricing range |
| Prospective client (warm, referred) | Find booking form fast, submit reference image + idea |
| Returning client | Check availability, see aftercare info |
| Stark (owner/operator) | Receive structured booking requests instead of scattered DMs; point people to a credible portfolio instead of a Facebook page |

### Primary user journey
Instagram/Facebook bio link → Home (hero + proof of skill) → Portfolio (style match) → Booking form → Confirmation → Manual follow-up by Stark.

---

## 3. Information Architecture

Single-page scroll-first structure with anchor navigation, not a traditional multi-page corporate site. Tattoo studio audiences browse like they browse Instagram — fast, visual, thumb-scroll. Deep pages (like full portfolio filtering) can exist but the primary path is one continuous page.

```
/ (Home)
 ├─ Hero
 ├─ About / Artist
 ├─ Portfolio (with category filtering, expands to /portfolio for full grid)
 ├─ Pricing Guide
 ├─ Reviews
 ├─ FAQ
 ├─ Booking (form)
 └─ Footer (contact, socials, location, hours)

/portfolio          → full filterable grid, lightbox
/portfolio/[piece]  → optional single-piece deep link (for sharing specific tattoos)
```

## 4. Visual & Brand Direction

### 4.1 Brand personality
Dark, precise, a little unsettling in a good way (horror realism), but not cluttered or "edgy teenager metal band" — Stark's actual work is highly technical and clean, so the site should feel **controlled darkness**, not chaos. Think: horror film title sequence, not heavy metal flyer.

Reference points to design toward (for tone, not to copy): A24 horror movie sites, tattoo flash sheet layouts, black & white film photography portfolios, brutalist portfolio sites for photographers/directors.

### 4.2 Typography — the most important section

**Do not use:** Poppins, Montserrat, Open Sans, Roboto, Lato, Inter (as a display font), Nunito, or any font that appears in default Canva/Wix/Framer AI templates. These read instantly as "generated."

Use a **two-font system**, both genuinely free/licensable, both underused in this exact category:

| Role | Font | Why |
|---|---|---|
| Display / Headlines | **Fraunces** (variable, use high-contrast "soft" optical size at wght 300–600, some instances italic) OR **Editorial New / PP Editorial New** if a licensed font is acceptable | High-contrast serif with an editorial, almost horror-poster feel when set large and tight. Reads as art-directed, not templated. Distinct from every "tech startup" sans-serif hero. |
| Body / UI / Labels | **Neue Montreal (PP)** or free alternative **General Sans** / **Space Grotesk** (used sparingly, not as brand voice) | Clean grotesk with personality in its details (look at the R, G, a) — legible at small sizes, doesn't compete with the display font. |
| Accent / stamp-like details (small doses only) | A single monospace, e.g. **JetBrains Mono** or **IBM Plex Mono**, used only for metadata like "SIZE — 6IN" "EST. 2021" "PLACEMENT: FOREARM" | Gives a tattoo-stencil / clipboard-form feel to small labels without going full "gothic metal font," which reads cheap. |

**Rules:**
- Never use a script/handwriting font for body copy. A single hand-lettered or etched-style font may be used *only* for the wordmark/logo treatment ("YOU THINK IT. I INK IT"), nowhere else.
- Headlines set large (60–140px desktop), tight letter-spacing, often uppercase in short bursts, but body copy stays warm and readable — no all-caps paragraphs.
- Avoid font pairings where both fonts are geometric sans — that's the #1 tell of a template.

### 4.3 Color system
Base palette must stay close to black & grey realism — the artist's own work is the brand.

| Token | Value | Use |
|---|---|---|
| `--ink-black` | #0A0A0A | Primary background |
| `--paper` | #F2F0EA (warm off-white, not pure #FFF) | Text on dark, light-section backgrounds |
| `--ash-grey` | #8B8B85 | Secondary text, dividers |
| `--blood` | #A31621 (desaturated, not neon red) | Single accent color — used sparingly: hover states, one accent per section max, small tags |
| `--skin-tone-neutral` | #C9BBA8 | Only in photography, never in UI chrome |

No gradients. No purple/blue "AI SaaS" tones anywhere. Flat black, flat paper white, one restrained red. This alone kills 80% of the "generic AI site" look.

### 4.4 No boxes, no drop shadows — what to use instead
Forbidden by default: `box-shadow` on cards, rounded-corner pastel icon tiles, glassmorphism, soft neumorphic buttons.

Use instead:
- **Hairline rules** (1px solid, `--ash-grey` at low opacity) to separate sections instead of card backgrounds.
- **Full-bleed images** with text overlaid directly on the photo (using a subtle black gradient scrim only where legibility requires it — not a decorative shadow).
- **Grain/halftone texture overlay** (a very subtle noise or halftone dot texture, 3–5% opacity) across dark sections to add tactile, print-like depth instead of shadow-based depth.
- **Asymmetric grid layouts**: portfolio and about sections should break out of a centered 3-column card grid. Use uneven column widths, images that bleed off the container edge, offset captions.
- **Underline / strike / bracket marks** as UI accents (e.g. `[ 01 ]`, a single red underline on hover) instead of button drop-shadows.
- Buttons: flat fill, hard or 2px corners (not fully rounded pills), a text label with a small arrow glyph, invert-on-hover instead of shadow-on-hover.

### 4.5 Layout uniqueness — concrete direction per section
- **Hero:** Full-bleed, full-viewport photo/video of a finished piece (or timelapse) as background. Wordmark small and fixed top-left, not centered. Headline "YOU THINK IT. I INK IT" set large, left- or bottom-aligned (not dead-center), with the CTA button placed off to the side, not stacked directly under the headline like a template.
- **Portfolio:** Masonry/asymmetric grid, not uniform square thumbnails. Category filter styled as a horizontal tab list with hairline underline active state, not pill buttons. Lightbox on click, swipe-through, minimal chrome.
- **About:** Split layout — large duotone portrait of Stark on one side, editorial-style pull quote + bio text on the other, with small mono-font metadata stats (years, tattoos done, studio est.) placed as a vertical sidebar rather than icon-boxes.
- **Pricing:** Presented as a simple structured table/list (like a menu board), not pricing "cards" with shadows and checkmarks — this isn't SaaS pricing.
- **Reviews:** Editorial pull-quote treatment for 2–3 standout reviews (large serif text) with a secondary condensed list/marquee of shorter quotes, rather than a uniform 3-card testimonial grid.
- **FAQ:** Accordion with a hairline top/bottom rule, plus-sign toggle that rotates to a minus — no card backgrounds.
- **Booking form:** Presented like a studio intake sheet — mono-font field labels, underline-style inputs (not boxed inputs with rounded borders and shadows), generous whitespace.

### 4.6 Anti-Generic Checklist (run this before shipping any screen)
- [ ] No Poppins/Montserrat/Inter/Roboto/Open Sans as the display typeface
- [ ] No `box-shadow` used for decorative card elevation
- [ ] No purple/blue gradient anywhere
- [ ] No perfectly centered hero-headline-subhead-button stack
- [ ] No uniform 3-card icon-box grid for services/features
- [ ] No fully rounded pill buttons as the primary CTA style
- [ ] At least one asymmetric/full-bleed layout moment per section
- [ ] Every accent color use is intentional and rare (max 1–2 red elements visible per screen)

## 5. Functional Requirements

### 5.1 Home / Hero
- Full-bleed hero image or looped muted video of tattooing/finished work
- Headline: "YOU THINK IT. I INK IT"
- Primary CTA: "Book a Session" → scrolls/links to Booking
- Secondary CTA: "View Portfolio"
- Sticky minimal nav (wordmark + 4–5 anchor links + Instagram/Facebook icons), transparent over hero, solid on scroll

### 5.2 About
- Portrait photo of Stark
- Short bio: 5+ years, Inkstarky Tattoo since Feb 2021, based Dagupan City, from Baguio
- Studio story (2–3 short paragraphs, written in Stark's voice, not corporate copy)
- Stat strip: Years active, tattoos completed (approx/optional), specialties

### 5.3 Portfolio
- Category filter: Realism, Horror, Portrait, Sleeve, Black & Grey, All
- Grid of finished-tattoo photography (high-res, consistent lighting/crop treatment across the set — this matters more than layout cleverness; commission or curate consistent photos)
- Click-through to lightbox with larger image, category tag, optional placement/size caption
- CMS-editable so Stark can add new pieces without a developer

### 5.4 Pricing Guide
- Structured tiers: Small / Medium / Large / Custom Quote
- Each tier: approximate size reference, starting price or price range, what's included (consult, one revision, etc.)
- Clear note: final quote depends on design complexity/placement — drives people to submit the booking form rather than expecting exact pricing upfront
- Deposit policy stated clearly here (ties to FAQ)

### 5.5 Booking / Appointment Form
Fields:
- Name (required)
- Phone (required)
- Email (required)
- Tattoo idea / description (required, textarea)
- Size (select: Small / Medium / Large / Not sure)
- Placement on body (text or select with common options)
- Preferred date(s) (date picker, or open text "flexible")
- Upload reference image(s) (multi-file upload, image only, size-limited)
- Consent checkbox (data use / studio policy)

Behavior:
- On submit: send structured email/notification to studio (and optionally to a Google Sheet or simple booking inbox), show on-screen confirmation, optionally trigger auto-confirmation email to client
- Basic spam protection (honeypot field or lightweight captcha)
- Mobile-first form — large tap targets, minimal typing where possible (selects/date pickers over free text)

### 5.6 Reviews
- Pull in / manually curate Facebook and Google reviews (via manual entry initially; API integration optional in Phase 2)
- Feature client photos alongside quotes where available (healed tattoo + testimonial pairing is high-converting for this niche)

### 5.7 FAQ
Accordion covering at minimum:
- Does it hurt / pain expectations
- Healing process & timeline
- How to prepare for a session
- Deposit policy (amount, refundability)
- Walk-ins vs. by-appointment
- Aftercare instructions (can also live as a downloadable/printable aftercare card)

### 5.8 Footer
- Studio name, address/general area (Dagupan City — exact studio address if public-facing), hours
- Instagram, Facebook links
- Contact (phone/email or "book via form above")
- Small print / copyright

---

## 6. Content Requirements
- **Photography is the product.** The single highest-leverage investment here is a consistent, well-lit photo set of finished/healed tattoos — flat lighting, neutral background, consistent crop. No layout decision matters more than photo quality/consistency.
- Bio and studio story copy should be written in first person, plain and direct — matches "YOU THINK IT. I INK IT" tone. Avoid marketing-agency voice ("elevate your ink journey").
- Aftercare content should be genuinely useful and specific (not generic filler) — this can be repurposed as a printed card handed to clients too.

---

## 7. Non-Functional Requirements

| Category | Requirement |
|---|---|
| Performance | LCP < 2.5s on mobile 4G; hero video/image must be compressed and lazy-loaded appropriately; use responsive image sizes for portfolio grid |
| Responsive | Mobile-first; primary traffic source is IG/FB bio-link on phones |
| Accessibility | Sufficient contrast even within the dark palette (verify `--ash-grey` on `--ink-black` meets at least AA for body text); form fields properly labeled |
| SEO | Local SEO priority — "tattoo artist Dagupan," "tattoo studio Pangasinan," structured metadata, Google Business Profile linkage |
| CMS | Portfolio and reviews should be editable by Stark without code (headless CMS or simple admin panel) |
| Hosting/Stack | Any modern stack works (e.g., a static site generator + headless CMS, or no-code builder capable of fully custom CSS/fonts — avoid builders that lock typography/layout into template presets) |

## 8. Phase Plan

**Phase 1 (MVP — launch target):**
Home, About, Portfolio (curated set, filter by category), Pricing Guide, Booking form, FAQ, Footer. Manually curated reviews section.

**Phase 2:**
Full `/portfolio` deep grid with per-piece pages (shareable links), live Google/Facebook review integration, aftercare downloadable PDF, email automation for booking confirmations, simple booking calendar/availability view.

**Phase 3 (optional):**
Client login area for aftercare check-ins, waitlist system, e-commerce for merch/gift cards if studio expands.
